== Введение

Данная лабораторная работа демонстрирует процесс создания и управления веткой разработки в системе контроля версий Git, а также последующее взаимодействие с удалённым репозиторием через веб-интерфейс GitHub. Работа выполнена в среде разработки Visual Studio с использованием языка C++ и микроконтроллерной платформы, в рамках проекта, связанного с обработкой сигналов от автомобильных устройств. Все этапы сопровождаются визуальной фиксацией через скриншоты, размещённые в директории `images/`.

=== 1. Создание новой ветки разработки

Для изоляции изменений, связанных с выполнением лабораторной работы №3, создана отдельная ветка `lab3` в локальном репозитории.

[.text-center]
image::image1.png[Создание ветки `lab3` в Git Desktop, width=600]

После создания ветки:

- Добавлен титульный лист в соответствии с требованиями к отчёту.
- Ветка `lab3` была запушена в удалённый репозиторий на GitHub.

[.text-center]
image::image2.png[Ветка `lab3` отображается в веб-интерфейсе GitHub, width=600]

> **Примечание**: GitHub — это веб-сервис для хостинга Git-репозиториев, предоставляющий интерфейс для управления ветками, pull request, код-ревью и интеграционными автоматизациями.

- Проведена проверка синтаксиса и валидация документа через локальный сборщик Asciidoctor.

Дополнительно использовались материалы лекций, техническая документация по микроконтроллерам и аналитические инструменты на основе ИИ для проверки корректности формулировок.

=== 3. Формирование Pull Request в GitHub

После завершения работы в ветке `lab3`, изменения были подготовлены к интеграции в основную ветку `main` через механизм Pull Request (PR).

Этапы:

1. Все изменения сохранены локально.
2. Выполнено `git push origin lab3` для отправки ветки на GitHub.
3. В веб-интерфейсе GitHub (https://github.com/your-username/your-repo) нажата кнопка **“Compare & pull request”**.
4. Указаны:
   - **Base**: `main`
   - **Compare**: `lab3`
   - Заголовок PR: `feat: Добавлен отчёт по лабораторной работе 3`
   - Описание: краткое резюме изменений, ссылки на скриншоты, ссылки на задачи (если есть)

[.text-center]
image::image4.png[Создание Pull Request в веб-интерфейсе GitHub, width=600]

---

=== 4. Ревью и одобрение Pull Request

После отправки PR преподаватель получил уведомление и провёл ревью через веб-интерфейс GitHub.

Этапы ревью:

1. Преподаватель перешёл на страницу Pull Request в браузере.
2. Проверил изменения в файлах, включая `.adoc` и изображения.
3. Убедился, что ветка `lab3` корректно сравнивается с `main` (различия отображаются в diff-представлении).
4. Назначен как reviewer (обязательный шаг для контроля качества).



[.text-center]
image::image5.png[Добавление reviewer (преподавателя) в Pull Request, width=600]

После проверки преподаватель оставил комментарии (если были) и, при отсутствии замечаний, нажал кнопку **“Approve”**.

[.text-center]
image::image6.png[Статус Pull Request: “Approved” , width=600]

=== 5. Слияние в основную ветку (Merge)

После получения одобрения:

1. Преподаватель нажал **“Merge pull request”**.
2. Выбрана опция **“Create a merge commit”** (для сохранения истории).
3. Подтверждён merge — ветка `lab3` интегрирована в `main`.

> После слияния ветка `lab3` может быть удалена (опционально) через кнопку **“Delete branch”** в интерфейсе GitHub.

== Вывод

В ходе лабораторной работы был реализован полный цикл разработки в рамках командного Git-Workflow:

- Создание изолированной ветки `lab3` для разработки отчёта.
- Редактирование документа в формате Asciidoctor в среде Visual Studio.
- Публикация изменений в удалённый репозиторий через GitHub.
- Формирование и ревью Pull Request через веб-интерфейс GitHub.
- Успешное слияние изменений в основную ветку `main` после одобрения преподавателем.

Процесс продемонстрировал корректное применение современных практик DevOps и контроля версий, соответствующих требованиям промышленной разработки. Все этапы зафиксированы скриншотами, расположенными в директории `images/`, что обеспечивает воспроизводимость и аудитируемость работы.
